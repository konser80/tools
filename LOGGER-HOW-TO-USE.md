# Как использовать в проектах

Подключение:
const logger = require('./logger');
const log = logger('trace', { dir: 'logs', hourly: false });

Используем:
log.trace(somedata, opts);
log.debug(somedata);

Важно:
Функция принимает лишь один аргумент для вывода, поэтому log.trace(arg1, arg2, arg3) не сработает.

# Когда какие методы применять

Аналогия: log.info() — это H1 на странице (редкие, важные события), log.debug() — H2 (основной поток), log.trace() — H3 (детали и сырые данные).

## trace

log.trace(raw) - это "нижний" уровень. Его можно вызвать перед вызовом "долгой" функции (например, перед внешним API запросом). Также через него можно логировать сырой ответ.
В консоль выводится синим цветом.

## debug

Через log.debug() стоит выводить результат действия уже отформатированный. Этот метод используем в большинстве случаев.
В консоль выводится серым цветом.

## info

log.info() используем для важных событий. Например - прилетел webhook или инициирован новый pipeline.
В консоль выводится белым цветом.

## warn, error

log.warn(), log.error() - стандартное использование.
